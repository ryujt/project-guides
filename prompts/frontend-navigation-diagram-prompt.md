# 프론트엔드 네비게이션 다이어그램 작성 프롬프트

## 목적

프로젝트의 모든 사용자 시나리오를 화면 전환 흐름으로 문서화한다.

## 지시사항

`/Users/ryu/projects/project-guides/guides/navigation-diagram-guide.md` 의 표기법과 작성 규칙을 반드시 따라서,
**사용자가 사용할 수 있는 모든 시나리오**를 통해 설명해줘.

### 세부 요구사항

1. **모든 진입점(Entry Point)을 식별한다.**
   * 라우터 정의, 페이지 컴포넌트, 최초 진입 화면을 먼저 파악한다.
   * 로그인/비로그인, 권한별 접근 등 사용자 상태에 따른 진입 흐름을 구분한다.

2. **시나리오 단위로 화면 전환을 작성한다.**
   * 각 시나리오는 사용자의 목적(예: 회원가입, 파일 업로드, 공유, 설정 변경 등)을 기준으로 나눈다.
   * 화면 전환(`Page --> Page`), API 호출(`Page --> (/api)`), 내부 로직(`(process)`), 분기(`: success`, `: error`)를 가이드 표기법대로 표현한다.
   * 각 코드 블록은 ```` ```navigation ```` 언어 태그로 감싼다.

3. **누락 없이 전체를 커버한다.**
   * 정상 흐름뿐 아니라 에러 처리, 검증 실패, 취소, 뒤로가기 등 예외 흐름도 포함한다.
   * 실제 코드(라우팅, 이벤트 핸들러, API 호출부)를 근거로 작성하고 추측으로 채우지 않는다.

4. **각 시나리오마다 간단한 설명을 덧붙인다.**
   * 시나리오 제목, 트리거 조건, 관련 화면/API를 명시한다.

### 표기 규칙 요약 (가이드 참조)

* 화면 이름: `Home`, `LoginForm` 등
* 백엔드 API: `(/signin)` — 소문자, 숫자, `/`, `_` 만 사용
* 내부 프로세스: `(validation)`, `(toast_error)`
* 메시지/데이터: 백틱으로 감싼 노드 `` `credentials` ``
* 분기: `: error`, `: success` (콜론 뒤 괄호 사용 금지)

## 산출물

* 시나리오별로 구분된 마크다운 문서 하나.
* 각 시나리오는 제목 + 설명 + ```` ```navigation ```` 다이어그램으로 구성.
