# 프론트엔드 상태 다이어그램 작성 프롬프트

## 목적

 각 화면에서 **화면 이동 없이** 내부적으로 벌어지는 상태 변화를 문서화한다.

## 지시사항

`/Users/ryu/projects/project-guides/guides/state-diagram-guide.md` 의 표기법과 작성 규칙을 반드시 따라서,
**화면 이동이 없는 상태에서 벌어지는 모든 일들에 대한 상태 변화**를 작성해줘.

### 세부 요구사항

1. **화면(Screen) 단위로 작성한다.**
   * 각 화면마다 별도의 섹션으로 상태 다이어그램을 작성한다.
   * 단, 상태 변화 없이 정보만 보여주는 단순 설명서/안내 화면 등 **프로세스가 지나치게 단순한 화면은 스킵**한다.

2. **화면 이동이 없는 상태 전이만 다룬다.**
   * 로딩 → 성공/실패, 입력 검증, 편집 모드 토글, 모달 열림/닫힘, 폼 제출 중/완료, 데이터 fetch 상태 등
   * 다른 화면으로 넘어가는 전환은 이 문서의 대상이 아니다(그건 네비게이션 다이어그램에서 다룬다).

3. **가이드 표기법을 따른다.**
   * 시작: `<s> --> (Ready)`
   * 상태 전이: `(Loading) --> (Loaded)`
   * 종료: `(Done) --> <e>`
   * 액션/프로세스는 괄호 없는 텍스트 노드로: `Save Log`
   * 분기 조건: `: Error`, `: Success` 등 `: Text` 형식
   * 각 코드 블록은 ```` ```state ```` 언어 태그로 감싼다.

4. **실제 코드를 근거로 작성한다.**
   * 컴포넌트의 state, 로딩/에러 플래그, 조건부 렌더링, 이벤트 핸들러를 확인하여 상태를 도출한다.
   * 추측으로 상태를 만들지 않는다.

### 표기 규칙 요약 (가이드 참조)

| 요소 | 입력 문법 | 시각적 표현 |
| --- | --- | --- |
| Start Node | `<s>` | 흰색 원 |
| State | `(Name)` | 둥근 사각형 |
| Action | `Name` (괄호 없음) | 직사각형 |
| End Node | `<e>` | 검은색 원 |

## 산출물

* 화면별로 구분된 마크다운 문서 하나.
* 각 화면 섹션은 제목 + 설명 + ```` ```state ```` 다이어그램으로 구성.
* 단순 화면은 제외한 사유를 짧게 남긴다(선택).
